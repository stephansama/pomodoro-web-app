# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
pnpm install        # install deps; esbuild build scripts are pre-approved via package.json#pnpm
pnpm dev            # dev server on http://localhost:5173
pnpm build          # tsc -b && vite build → dist/
pnpm preview        # serve dist/ on http://localhost:4173
pnpm lint           # tsc --noEmit only — no ESLint configured
```

Use **pnpm exclusively** — never `npm` or `npx`. There is no test suite yet. Node ≥ 20 required.

`src/routeTree.gen.ts` is generated by `@tanstack/router-plugin/vite` on first `pnpm dev` / `pnpm build`. It's gitignored. If you add a new route file under `src/routes/`, `tsc -b` will fail with a misleading "argument of type '\"/foo\"' is not assignable to parameter of type undefined" error until the dev server runs once to regenerate the tree.

## Source-of-truth precedence

When the three docs disagree, the higher one wins:

1. **`~/Downloads/Tomato.html`** (or its decoded `/tmp/tomato_decoded.html` if rebuilt) — the original frontend-design HTML. Owns the visual truth: tokens, component CSS, motion.
2. **`PLAN.md`** — owns the functional scaffold: data model, route names, timer behavior, PWA wiring.
3. **`DESIGN.md`** — owns principles + a11y + keyboard shortcuts + empty-state coverage. Used to fill gaps the design HTML doesn't speak to.

The repo's `src/styles/components.css` is a verbatim port of `Tomato.html`'s component CSS. **Don't restyle by editing component JSX class names** — change the CSS instead so the design contract stays intact.

## Architecture

### Phase-aware theming (the central design idea)

`html[data-phase="focus|short|long"]` is the master switch. `src/styles/tokens.css` defines:

```css
html[data-phase="focus"] { --accent: #BA4949; --accent-soft: var(--focus-soft); }
html[data-phase="short"] { --accent: #38858A; --accent-soft: var(--break-soft); }
html[data-phase="long"]  { --accent: #397097; --accent-soft: var(--long-soft); }
```

Every accent-driven surface — nav active link, phase tabs, primary button, timer ring stroke, task active border, Recharts bars/line — reads `var(--accent)` directly or through a `bg-primary`-style Tailwind utility. `src/hooks/use-phase.ts` is the only thing that should write this attribute (it also updates the `theme-color` meta tag). Never hardcode phase hex colors in components.

`use-theme.ts` does the same for `html[data-theme="light|dark"]`, syncing to `settings-store.theme` and to the OS `prefers-color-scheme` when `theme === 'system'`.

### Tailwind v4 → design-token bridge

`src/styles/index.css` declares `@theme inline { --color-primary: var(--accent); ... }` which makes shadcn utility classes (`bg-primary`, `text-card-foreground`, etc.) resolve to the design tokens, and therefore re-tint per phase. Don't bypass this with hex literals.

### shadcn primitives are restyle wrappers

`src/components/ui/*` uses Radix for behavior (focus traps, keyboard, ARIA) and the design's CSS class names (`.switch`, `.herotabs`, `.htab`, `.pbtn`, `.ctrlbtn`, …) for visuals. Adding a new primitive: write a thin Radix wrapper, then add component CSS to `src/styles/components.css` — don't reach for shadcn's stock Tailwind class strings.

### Timer engine is split in two

Two hooks share the same store but have different purposes — don't merge them:

- **`useTimerHeartbeat()`** (in `src/hooks/use-timer.ts`, mounted ONLY in `src/routes/__root.tsx`) — global 250ms tick. When `status === 'running' && remainingMs <= 0`, it records the `Session` to Dexie, fires the chime + Notification, advances phase (respecting `longBreakInterval` and `autoStart*` settings), and re-enters running if auto-start. Re-entry is guarded by a `handlingRef` ref to prevent double completion under React StrictMode's double-effect.
- **`useRemaining()`** (mounted in `src/routes/index.tsx`) — local 250ms tick that returns ms remaining for the Timer UI to re-render. Drives the ring stroke and numerals. No side effects.

State is timestamp-based, not interval-based: `remainingMs(state)` in `src/stores/timer-store.ts` derives from `Date.now() - startedAt - accumulatedPausedMs`. This is why a mid-session refresh resumes correctly — the Zustand `persist` middleware rehydrates `startedAt` and the math just keeps working.

Imperative actions are exported as plain functions from `use-timer.ts`: `startPhase`, `selectPhase`, `stopTimer`, `skipPhase`. They write directly to `useTimer.setState` and don't need component context. Prefer these over wiring custom event handlers.

### Fullbleed shell

`src/routes/__root.tsx` adds the `fullbleed` class on `.site` **only** when the user is on `/` AND `timer.status !== 'idle'`. The CSS in `components.css` then inverts every internal contrast rule via `.site.fullbleed .X` selectors (translucent tabs, white-on-accent button, white nav links, etc.). If you add a new component that appears during a running session, give it a matching `.site.fullbleed .your-class` rule or it will look broken when the room dims.

### Persistence

Two layers, intentionally separate:

- **Dexie / IndexedDB** (`src/db/`) — durable user content: tasks + sessions. Schema in `dexie.ts`. Query helpers (totals, daily/weekly buckets, per-task split) in `sessions.ts`. Tasks are **soft-deleted** via `archivedAt` so historical sessions keep their reference; hard delete via `deleteTaskHard` detaches sessions instead of cascading.
- **Zustand `persist` / localStorage** (`src/stores/`) — ephemeral UI state that needs to survive a refresh: the mid-session timer (`tomato-timer`) and user settings (`tomato-settings`). Per-task totals and stats are computed on demand from Dexie, never denormalized into Zustand.

`dexie-react-hooks`' `useLiveQuery` is the reactivity bridge — any component reading from Dexie should use it so writes show up immediately.

### Routing

TanStack Router with file-based routes under `src/routes/` and `autoCodeSplitting: true` in `vite.config.ts`. Each route ships as its own chunk. `Tabs`, `Stats`, `Settings` lazy-load. `__root.tsx` is the only place where global hooks (`useTheme`, `usePhase`, `useTimerHeartbeat`, `usePwaUpdate`, `useKeyboardShortcuts`) should be mounted.

### PWA

`vite-plugin-pwa` with `registerType: 'autoUpdate'` and `devOptions.enabled: false` (so the SW only runs in production builds). `src/hooks/use-pwa-update.ts` imports `virtual:pwa-register` and shows a Sonner toast with a `Reload` action when a new SW is waiting. `src/components/pwa/InstallPrompt.tsx` captures `beforeinstallprompt` and respects a 14-day "not now" dismissal in localStorage. The `theme-color` meta is mutated at runtime by `use-phase.ts` so installed-app chrome retints with each phase.

### Keyboard shortcuts

Defined once in `src/hooks/use-keyboard-shortcuts.ts`, mounted in `__root.tsx`. Bypassed when the target is an input / textarea / contenteditable (see `isTypingInForm`). Bindings: `Space` start/pause/resume, `S` skip, `Esc` stop (confirm), `1·2·3` jump phases, `?` opens the shortcut sheet.

## Conventions

- Path alias `@/*` → `src/*` (set in `tsconfig.app.json` and `vite.config.ts`). Use it for cross-module imports; relative paths only within the same folder.
- Lucide for all icons; design specifies stroke width 1.75 and 20 px by default. Don't switch to filled glyphs for active states — color and background carry that information.
- Timing values come from `--ease` and `--ease-emph` in `tokens.css`. Don't introduce new cubic-béziers unless you also add a token.
- Sonner is the only toast system — don't add a competing one. Its container is mounted in `__root.tsx`.
- The `public/__seed.html` localStorage seeder used during build is gone — don't reintroduce it without a `dev`-only guard.
